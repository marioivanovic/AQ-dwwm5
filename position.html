Position

Static (Normal)

Cas d'utilisation : - Flux normal du document pour la plupart des éléments - Paragraphes de texte - Images inline - Éléments de formulaire standard

Relative (Un peu bougé)

Cas d'utilisation : - Ajustements mineurs de position sans affecter les autres éléments - Conteneur pour des éléments en position absolute - Création de superpositions légères (z-index)

Absolute (Où tu veux)

Cas d'utilisation : - Superposition d'éléments (modals, tooltips) - Icônes ou badges positionnés sur d'autres éléments - Éléments de navigation flottants - Positionnement précis d'éléments dans un conteneur

Fixed (Collé)

Cas d'utilisation : - Barres de navigation qui restent visibles lors du défilement - Boutons "Retour en haut" - Bannières d'annonces ou de cookies - Chat widgets fixes
Jouet

Fais défiler cette boîte pour voir le jouet collant en action !

Il reste en haut quand tu fais défiler, mais seulement dans cette boîte.

Cas d'utilisation : - En-têtes de section qui restent visibles lors du défilement - Navigation secondaire dans une longue page - Éléments de publicité qui suivent le défilement dans une certaine limite - Table des matières qui reste visible

Le jouet "Fixed" reste collé en bas à droite de la fenêtre, même quand tu fais défiler la page.

Par défaut, le contexte de positionnement pour tous les éléments est le viewport (la fenêtre du navigateur).
Cela signifie que si vous positionnez un élément en absolute sans un parent positionné, il se positionnera
par rapport aux bords de la fenêtre du navigateur

Lorsqu'un élément est positionné avec relative, absolute, ou fixed, il crée un nouveau contexte de
positionnement pour ses enfants. Cela signifie que :
Pour position: relative :
L'élément lui-même reste dans le flux normal du document.
Il devient un point de référence pour les éléments enfants positionnés en absolute.
Pour position: absolute :
L'élément est retiré du flux normal du document.
Il devient un point de référence pour ses propres enfants positionnés en absolute.
Il se positionne par rapport à son ancêtre positionné le plus proche (pas statique),
ou le viewport si aucun ancêtre n'est positionné.
Pour position: fixed :
L'élément est retiré du flux normal du document.
Il se positionne toujours par rapport au viewport.
Il crée un nouveau contexte de positionnement pour ses enfants, mais ceux-ci se positionnent
par rapport à l'élément fixed, pas au viewport.
